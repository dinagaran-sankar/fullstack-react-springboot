2.ReadyStateChange()
====================
Fires whenever the document.readyState property changes.
"loading" – The document is still loading.
"interactive" – The DOM is fully parsed, but sub-resources (images, stylesheets) may still be loading.
"complete" – The page and all resources are fully loaded.

document.addEventListener('readyStateChange',()=>{
   if(document.readyState='interactive')
   {
   console.log("DOM is ready (like DOMContentLoaded).");
   }
   else if (document.readyState=='complete')
   {
     console.log("Everything loaded (like window.onload).");
   }
})
1.Can detect multiple stages of loading.
2.Works in older browsers (including old IE).

Asynchronous
============
Prevents browser freeze which is critical for user expereince
js execute top to bottom left to right line by line code

why we use Asynchronous
======================
1.Some tasks take time to finish (network requests, timers, user input).
2.To stay responsive, JavaScript can use async programming.
3.Asynchronous flow refers to how JavaScript allows certain operations to run in the background and let their results be handled when they are ready.
4.If JavaScript waited for these tasks, the page would freeze.
5.Asych code lets the rest of the program continue to run.
6.Async code does not run immediately.

SetTimeout() is a kind of asynchronous operation
Callback() is a kind of asynchronous operation once runs after something has finished.

Promise
=======
It Provides result,
1.A Promise object represents the completion or failure of an asynchronous operation.
2.A promise represents a value that will be available later.
3.A promise is a container for a future result.
4.The result can be a value or an error.
5.By default promise handled error .

A Promise can be in one of three exclusive states:

pending	 -   operation started (not finished) - The initial state; the operation has started but is neither fulfilled nor rejected.
rejected -	operation failed - The operation completed successfully, and a value is available.
fulfilled -	operation completed - The operation failed, and a reason (error) is available.

Syntax
======
let myPromise = new Promise(function(resolve, reject) {
// Code that may take some time
resolve(value); // when successful
reject(value);  // when error
});

const promises = new promises((resolve,reject)=>{
   resolve(value);
   reject(value);
});

.catch()- reject handled

.finally()

Why we use Promise
===================
Single Thread Language - javascript
reduce code several callback() function
if we integrated 3 rd party js file then there is some network delay to overcome this promise asynchronous operation we are using.
Better error handling: .catch() makes handling errors straightforward.
Chaining: Promises allow sequential execution of async tasks using .then().
Integration with async/await: Modern JavaScript uses Promises under the hood for async/await, making async code look synchronous.
modern support
